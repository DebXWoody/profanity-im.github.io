Why another console based XMPP client?

Profanity was originally written for two main reasons. Firstly, the author wanted something similar to Irssi, but with a focus on XMPP for use in his workplace. Secondly, it was a bit of fun and a learning exercise.

Profanity is still a young project and there are plenty of good alternatives out there (although of course we'd love you to use Profanity), see the list at xmpp.org.

Will Profanity work with MSN/Yahoo/AIM/IRC etc?

No, Profanity is purely an XMPP chat client.

Can I connect to more than one account at a time?

No, the recommended way is to use multiple terminal windows, or GNU Screen.

Why does the F1 key open terminal help rather than going back to the main console window?

Some terminals have a key binding for F1 to open help. You can use the alt-num keys to change windows, or if you prefer to use the F keys, you'll need to change the terminal setting to open help with another key.

How can I select text from Profanity to copy to the clipboard?

You have to hold the shift key down to select text with the mouse.

Why don't the desktop notifications work for me?

Desktop notifications are currently only supported on Linux (for which you need libnotify available during compile), and Windows using the basic win32 balloon API.

OS X desktop notifications should be implemented in a future release.

I've found a bug/have a feature request who do I tell?

If you have a github account, you can log an issue on the issue tracker.

Alternatively you are welcome to post on the mailing list.

Finally you can email the project maintainer at boothj5web@gmail.com.

I'd like to contribute, what should I do?

You can fork the project on Github and try to fix an issue or add a feature.

The XMPP standard has many features and extensions that are not yet implemented in Profanity, so you could try to tackle one of those.

Feel free to post on the mailing list if you have questions, or just email the project maintainer at boothj5web@gmail.com if you don't know where to start.

Why after upgrading from 0.1.9 to a later version isn't my configuration being picked up?

Configuration file locations have been changed to be compliant with the XDG Base Directory Specficiation and some changes have been made to the format.

Such non backwards compatible changes should be very rare in future but were required to make things more extensible from now on.

Details of the new configuration file locations can be found here and the commands used to manipulate the settings here.

The simplest way to get your configuration back (there were only a few options in 0.1.9) is to start Profanity after upgrading and use the commands to set your desired options.

Since basic colour themes are now supported, colours are no longer specified in the main configuration file, so you will need to create a theme as shown in the link.
