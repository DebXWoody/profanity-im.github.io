Use Profanity

The easist way you can help out is just by using Profanity and sending any feedback you have to the mailing list

Any feedback is welcome, although please keep it polite, and refrain from using profanity (pun intended)

If you prefer not so use the mailing list, feel free to send a mail directly to boothj5web@gmail.com

Report Bugs

Should you come across a bug, the preferred method is to log it at the github issue tracker

If you do not have a github account, you can still post the issue at the mailing list, or email boothj5web@gmail.com directly.

The more information you can supply the better chance we'll have of recreating the issue. Your OS/Distribution/Environment etc.

A list steps to reproduce the issue can be very helpful.

Running Profanity with the following argument will set debug logging, (remember to remove any sensitive information before sending the logs):

$ profanity -l DEBUG

The logs can be found as per the files page, usually: 

~/.local/share/profanity/logs/profanity.log

For those that know how, providing backtraces of coredumps can also be useful for crashes.

If you don't have time for any of the above, a simple description of the problem would still be preferred over no bug report at all.

Help with the Website

The website often has small errors or omissions in documentation etc. The website source is also available at github so feel free to send pull requests there.

Again, if you don't have time, a quick mail to boothj5web@gmail.com would be welcome.

Testing

If you are one of those who enjoys trying to break software, we'd love you to break profanity and tell us how you did it!

Contribute Code

Code contributions are always very welcome. From the Small (spelling mistakes in the built-in help) to the large (completely new XMPP features).

Documentation on the current code is pretty much non-existent, and needs to be improved.

Currently there is one maintainer, another regular contributor, and many one off contributors.

If you would like to get involved, you can submit a pull request, or if you just want to get involved but don't know how, email boothj5web@gmail.com.

The original author knew next to nothing about coding real applications in C, XMPP, or building and installing software on Linux when starting the project (which probably shows!), so if you just want to join in and learn, you are also welcome.

Specific Help Required

The following areas are currently where some specific expertise would help:

OSX development and testing - Whilst Profanity does work on OSX, there are some quirks in the build, and currently there is no developer on the project specifically taking care of this area.

Release process - The current release process is fairly manual. Using the autotools as much as possible to automate this process would be useful (see next point).

Distribution packages - The install process is a mix of an install script and some distribution specific files. Any expertise in various distributions would be most welcome.

OTR support - Off The Record messaging is currently in a development branch, but hasn't yet had much time spent on it.
