/account

Manage local accounts. A local account has the following properties:

name - The name for the account.
jid - The Jabber ID of the account.
server - The chat server to use, if different to the domainpart of the Jabber ID.
port - The port used to connect to the server, if not the default.
resource - The name of the resource to connect with.
password - The account password.
status - The presence status to be used when connecting with this account.
status priority - A priority associated with each presence status.
muc service - The default conference service to use when joining chat rooms.
muc nickname - The default nickname to use when joining chat rooms.
otr setting - The account wide OTR policy.

Accounts may be enabled/disabled, the only difference being that enabled accounts are used for autocompletion.

See Advanced presence, resources and priorities for an overview of account presence priorities.

When connecting with the /connect command, if an account does not currently exist with the connection details, one will be created using the Jabber ID as the name.

You may rename this account at a later point if desired, see usage below.

For details of where accounts are stored, see the help on file locations.

Basic usage:

/account command [account]

Arguments:

The command may be:

list List all accounts.
add Create an account, passing the account name as an argument.
show Show details and all settings of the specified account.
enable Enable the specified account.
disable Disable the specified account.

Rename usage:

/account rename oldname newname

Arguments:

oldname The existing account to rename.
newname The new account name.

Setting properties:

/account set account property value

Arguments:

account The account name you wish to edit.
property One of jid, server, port, resource, password, muc, nick, status or a presence (online, chat, away, xa, dnd).

Values for each property are described below.

jid The Jabber ID to use for the account, e.g. me@myservice.org. Defaults to the account name.
server The server to connect to for the account, e.g. talk.google.com. Optional with no default.
port The port used when connecting to the chat server, e.g. 5355. Optional with a default of 5222.
resource The resource name to use when connecting with this account, e.g. laptop. Defaults to profanity.
password The password for the account. Note, the password will be stored in plaintext. Optional with no default.
muc The default conference service for this account, e.g. conf.server.org. Defaults to 'conference.<domainpart>'.
nick The default nickname to use when joining chat rooms, e.g. eddie. Defaults to the localpart of the Jabber ID.
otr The account OTR policy, setting may be one of manual, opportunistic or always. See the OTR page for more information on OTR policies.
status The presence status to use when connecting with this account. Values are online, chat, away, xa, dnd and last, where the special value last always uses the last presence that this account was connected with.
online, chat, away, xa, dnd A valid priority (between -128 and 127) to use for the specified presence status.

Clearing properties:

/account clear account property

Arguments:

account The account name you wish to edit.
property The only property which may be cleared is password.

Example:

/account add work
/account set work jid myuser@mycompany.com
/account set work server talk.google.com
/account set work port 5432
/account set work resource laptop
/account set work status away
/account set work chat 20
/account set work online 10
/account set work away 5
/account set work xa 0
/account set work dnd -1
/account rename work gtalk

You can then log into the above account using /connect gtalk.

The talk.google.com server will be used to connect rather than mycompany.com, using port 5432

The resource associated with this account when logged in will be 'laptop'.

Each time you log in, your status will be away

When you change status to 'chat' by using the /chat command for example, the priority for Profanity will be set to 20.

/alias

Add command aliases. This command can be used to create shortcuts and avoid typing the same command over and again.

For example, to add a command alias /waf which will alias to /who available friends:

/alias add waf /who available friends

To remove an alias:

/alias remove waf

To list all current aliases:

/alias list

/autoaway

Configure profanity to send either a last activity presence, or an away status change when you have been idle for a given amount of time.

If built in an X-Windows environment, with the libxss library available (this is the case with most Linux distributions), the desktop idle time will be used, otherwise the last input to the profanity window is used.

Usage:

/autoaway command value

Arguments:

command One of mode, time, message, check, see below for arguments to each.
value mode - Values are away, idle, off.
time - Time in minutes before sending presence change.
message - Optional message, or off for no message.
check - Automatically set status to online when input received, on or off.

Defaults:

mode is set to off.

time is set to 15 minutes.

message is off.

check is set to on.

Examples:

To set profanity to send your idle time (with your presence remaining online) to contacts after a period of 30 minutes, with a message of "I'm not around", and report when you are no longer idle:

/autoaway mode idle
/autoaway time 30
/autoaway message I'm not around
/autoaway check on

To set profanity to change your status to 'away' to contacts after a period of 1 hour, with no message, and not automatically update status when you are no longer idle:

/autoaway mode away
/autoaway time 60
/autoaway message off
/autoaway check off

/bookmark

Manage MUC chat room bookmarks, including autojoining rooms on login.

Usage:

/bookmark list

List all current bookmarks. Rooms that have been joined will be highlighted and the window number displayed.

/bookmark add|update room [nick value] [password value] [autojoin on|off]

Add a bookmark, or update an existing bookmark. The room argument is mandatory, the options are:

nick The nickname to use for entering the room, if different to the account default.
autojoin Whether or not to join the room when this account connects.
password The password to use if the room is private. This option is not recommended as the password may be stored in plaintext on the server. 
/bookmark remove room

Remove a bookmark.

/bookmark join room

Join the bookmarked room using the settings associated with the bookmark.

/bookmark

When in a chat room, using the command with no arguments will create a bookmark for the current room, with the current nickname, and set autojoin to on.

/mouse

Enable/disable mouse handling.

When enabled, Profanity takes control of mouse actions, allowing the mouse wheel to roll the chat window contents for example.

Enabling mouse handling may require shift to be held down to select text.

This feature is currently experimental, problems have been reported where the UI locks up after certain mouse click combinations, until a key is pressed or another mouse event received.

Usage:

/mouse on|off

Default:

Set to off.

/notify

Configure desktop notifications, profanity must be compiled with libnotify for desktop notifications to work.

Usage:

/notify type [value|setting value]

Regular chat messages:

/notify message on|off Enable/disable regular chat message notifications.
/notify message current on|off Enable/disable regular chat message notifications for the current window.
/notify message text on|off Show/hide message text in notifications for regular chat messages.

Chat room messages:

/notify room on|off|mention Enable/disable chat room message notifications, or enable only when nickname mentioned in message.
/notify room current on|off Enable/disable chat room message notifications for the current window.
/notify room text on|off Show/hide message text in notifications for chat room messages.

Typing:

/notify typing on|off Enable/disable notifications when a contact is typing.
/notify typing current on|off Enable/disable notifications when a contact is typing for the current window.

Invites:

/notify invite on|off Enable/disable notifications for chat room invites.

Subscription requests:

/notify sub on|off Enable/disable notifications for contact subscription requests.

Reminders:

/notify remind [seconds] Set the period for desktop reminder notifications of unread messages. Use 0 to disable reminders.

Examples:

To enable desktop notifications when new regular messages arrive:

/notify message on

To show message text for regular message notifications:

/notify message text on

To notify on room messages only when nickname mentioned:

/notify room mention

To disable chat room notifications when the chat room window is visible:

/notify room current off

To enable desktop notifications when a contact is typing:

/notify typing on

To enable desktop notifications when chat room invites are received:

/notify invite on

To enable desktop notifications when a subscription request is received:

/notify sub on

To get desktop notifications reminding you of any of the above every 30 seconds:

/notify remind 30

/otr

Off The Record message encryption.

See OTR Message Encryption for an overview of Off The Record messaging.

Usage:

/otr gen

Generate a key for the current logged in account.

/otr myfp|theirfp

myfp - Show the fingerprint for your key in the current window.
theirfp - Show the fingerprint of a contact's key, you must be in an OTR session with the contact.

/otr start [contact]

Start an OTR session. If you are already in a chat window the [contact] may be omitted.

/otr end

End an OTR session with the current recipient in a chat window.

/otr secret secret

Start, or respond to shared secret authentication passing the secret as an argument.

/otr question question answer

Start question answer authentication, passing as arguments the question and expected answer.

/otr answer answer

Respond to question answer authentication, passing your answer as an argument.

/otr trust|untrust

Manage the trust level of a contact, you must be in an OTR session to change trust.

trust - Mark the current contact as trusted.
untrust - Mark the current contact as untrusted.

/otr policy setting [contact]

Manage the global or per contact OTR policy.

If the contact is omitted, the global setting will be updated. Possible settings are:

manual The default, OTR session must be manually started with /otr start.
opportunistic Upon sending a message, profanity will attempt to start an OTR session with the recipient.
always You will receive an error message if you attempt to send an unencrypted message. 
/otr log setting

Configure how OTR messages are logged, setting may be one of:

on Enable plaintext logging of OTR messages in chat logs.
off Disable logging of OTR messages.
redact Log OTR messages but replace the contents with the string [redacted]. 
/otr warn on|off

Configure whether an [unencrypted] warning appears in the titlebar for message sessions that are not encrypted.

By default this setting will be set to on

/otr libver

Show which version of the libotr library is being used.

/wins

Manage active virtual windows.

Calling with no arguments will list all active virtual windows, with information about their contents.

To shuffle the virtual windows so there are no gaps between them, use:

/wins tidy

To close any windows with no unread messages, and then shuffle so there are no gaps:

/wins prune

Two windows may be swapped by using the following command:

/wins swap n m

Where n and m are window positions. m may be an empty position.
