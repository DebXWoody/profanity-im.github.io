/account

Manage local accounts. A local account consists of a name, a Jabber ID and an optional server property.

Accounts may be enabled/disabled, the only difference being that enabled accounts are used for autocompletion.

When connecting with the connect command, if an account does not currently exist with the connection details, one will be created using the Jabber ID as the name.

You may rename this account at a later point if desired, see usage below.

For details of where accounts are stored, see the help on file locations.

Basic usage:

/account command [account]

Arguments:

The command may be:

list List all accounts.
add Create an account, passing the account name as an argument.
show Show details and all settings of the specified account.
enable Enable the specified account.
disable Disable the specified account.

Rename usage:

/account rename oldname newname

Arguments:

oldname The existing account to rename.
newname The new account name.

Setting properties:

/account set account property value

Arguments:

account The account you wish to edit.
property One of jid, server, status or a presence online, chat, away, xa, dnd
value jid - The value is the Jabber ID to use for the account.
server - The value is the server to connect to.
resource - The name to be used as the resource for Profanity, the default is 'profanity'.
status - The presence to use when logging in to this account. A value of last will use the presence at the time the previous logged in session logged out
presence - The value is the priority to use for this presence.

Example:

/account add work
/account set work jid myuser@mycompany.com
/account set work server talk.google.com
/account set work resource laptop
/account set work status away
/account set work chat 20
/account set work online 10
/account set work away 5
/account set work xa 0
/account set work dnd -1
/account rename work gtalk

You can then log into the above account using /connect gtalk.

The talk.google.com server will be used to connect rather than mycompany.com.

The resource associated with this account when logged in will be 'laptop'.

Each time you log in, your status will be away

When you change status to 'chat' by using the chat command for example, the priority for Profanity will be set to 20.

/autoaway

Configure profanity to send either a last activity presence, or an away status change when you have been idle for a given amount of time.

If built in an X-Windows environment, with the libxss library available (this is the case with most Linux distributions), the desktop idle time will be used, otherwise the last input to the profanity window is used.

Usage:

/autoaway command value

Arguments:

command One of mode, time, message, check, see below for arguments to each.
value mode - Values are away, idle, off.
time - Time in minutes before sending presence change.
message - Optional message, or off for no message.
check - Automatically set status to online when input received, on or off.

Defaults:

mode is set to off.

time is set to 15 minutes.

message is off.

check is set to on.

Examples:

To set profanity to send your idle time (with your presence remaining online) to contacts after a period of 30 minutes, with a message of "I'm not around", and report when you are no longer idle:

/autoaway mode idle
/autoaway time 30
/autoaway message I'm not around
/autoaway check on

To set profanity to change your status to 'away' to contacts after a period of 1 hour, with no message, and not automatically update status when you are no longer idle:

/autoaway mode awat
/autoaway time 60
/autoaway message off
/autoaway check off

/autoping

Configure profanity to send a ping to the chat server on a regular basis. This can sometimes be required to keep a connection from closing.

Usage:

/autoping seconds

Arguments:

seconds The interval in seconds between sending pings to the server. A value of 0 will diable automatic server pings.

Default:

Set to 0 (disabled) by default.

The full jid, is of the form localpart@domainpart/resource, e.g.:

/connect

Log in to an XMPP chat server.

Usage:

/connect account [server]

Arguments:

account - Either the Jabber ID you wish to log in as, e.g. user@server.org, or the account name of a previously created account, see the account command.

server - Optional server to connect to if the chat service is hosted at a different domain to the domain part of the Jabber ID.

The /connect command automatically creates an account using the Jabber ID as it's name if one does not exist already. See the account command for more information.

/mouse

Enable/disable mouse handling.

When enabled, Profanity takes control of mouse actions, allowing the mouse wheel to roll the chat window contents for example.

Enabling mouse handling may require shift to be held down to select text.

Usage:

/mouse on|off

Default:

Set to on.

/msg

Send a message.

Normal usage:

/msg contact [message]

Arguments:

contact The full Jabber ID of the contact you wish to message, e.g. contact@server.org.
message The message to send, does not need to be quoted, if not supplied, a new chat window is opened with the contact, or an existing one becomes visible.

Chat room usage:

When in a chat room, the message command can be used to send a private message:

/msg nick [message]

Arguments:

nick The room participants nickname, use double quotes if the nickname contains spaces.
message The message to send, does not need to be quoted, if not supplied, a new chat window is opened with the contact, or an existing one becomes visible.

/notify

Configure desktop notifications, profanity must be compiled with libnotify for desktop notifications to work.

Usage:

/notify type value

Arguments:

type One of message, typing, remind, see below for arguments to each.
value message - Values are on or off.
typing - Values are on or off.
remind - Seconds between showing a desktop notification reminder of unread messages.

Defaults:

message is set to off.

typing is set to off.

remind is set to 0 seconds (disabled).

Examples:

To enable desktop notifications when new messages arrive:

/notify message on

To enable desktop notifications when when a contact is typing:

/notify typing on

To get desktop notifications reminding you that you have unread messages every 30 seconds:

/notify remind 30

/sub

Manage presence subcriptions.

Usage:

/sub command [contact]

If used whilst in chat with a contact, the contact parameter may be omitted and the current contact will be used.

Arguments:

command request - Requests presence subscriptions from a contact.
allow - Approve a request from a contact to see your presence information.
deny - Deny a request from a contact, or remove them from your previously approved list.
sent - Show a list of sent presence subscriptions for which the recipient has not yet responded.
received - Show a list of received presence subscriptions to which you have yet to respond to.
contact The contact for whom you are managing subscriptions, may be omitted if currently in chat with a contact.

Default:

/who

List members from your roster in various presence states. Whilst in a chat room, list room members.

Usage:

/who [presence]

Arguments:

presence offline - Show all offline contacts.
chat, dnd, away, xa - Show contacts with the specified presence.
online - Show all online contacts, includes those who are 'chat', 'away', 'xa', 'dnd'.
available - Show all contacts with a presence of 'online', 'chat'.
unavailable - Show all contacts with a presence of 'offline', 'away', 'xa', 'dnd'.

Omitting this argument will show all contacts in your roster, or in the room.
