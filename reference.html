Basic commands

/about

Show version and license information.

/close

Close a chat or chat room window.

/connect

Log in to an XMPP chat server.

Usage:
/connect login
Arguments:

login - The Jabber ID you wish to log in as, e.g. user@server.org.

/disconnect

Log out of the currently logged in session.

/help

Show help information.

Usage:
/help [list|area|command]
Arguments:
list List all commands.
area One of basic, presence, settings, navigation, shows summary help for commands in that area.
command The specific command on which you want detailed help.

/info

Show status information for a contact in your roster.

Usage:
/info contact
Arguments:
contact The full Jabber ID of the contact you are interested in, e.g. contact@server.org.

/join

Join a chat room.

Usage:
/join room [nick]
Arguments:
room The Jabber ID of the chat room you wish to join, e.g. room@conference.server.org.
nick Optional nickname to use in the room. If this parameter is not supplied, the beginning part (before the '@') of your Jabber ID will be used.

/msg

Send a message.

Normal usage:
/msg contact [message]
Arguments:
contact The full Jabber ID of the contact you wish to message, e.g. contact@server.org.
message The message to send, does not need to be quoted, if not supplied, a new chat window is opened with the contact, or an existing one becomes visible.
Chat room usage:

When in a chat room, the message command can be used to send a private message:

/msg roommember [message]
Arguments:
roommember The full Jabber ID of the chat room member, including chat room and nickname, e.g. room@conference.server.org/nick
message The message to send, does not need to be quoted, if not supplied, a new chat window is opened with the contact, or an existing one becomes visible.

/nick

Change nickname when in a chat room.

Usage:
/nick nickname
Arguments:
nickname The new nickname you wished to be known as in the chat room.

/prefs

Show current configuration settings.

Usage:
/prefs [area]
Arguments:
area Show preferences in a specific area, one of ui, desktop, chat, log, conn, presence. If no argument is supplied, all preferences are shown.

/quit

Quit Profanity.

/sub

Manage presence subcriptions.

Usage:
/sub command [contact]

If used whilst in chat with a contact, the contact parameter may be omitted and the current contact will be used.

Arguments:
command request - Requests presence subscriptions from a contact.
allow - Approve a request from a contact to see your presence information.
deny - Deny a request from a contact, or remove them from your previously approved list.
sent - Show a list of sent presence subscriptions for which the recipient has not yet responded.
received - Show a list of received presence subscriptions to which you have yet to respond to.
contact The contact for whom you are managing subscriptions, may be omitted if currently in chat with a contact.

/tiny

Use the TinyURL service to send a tiny URL link, may only be used when in chat with a contact.

Usage:
/tiny url
Arguments:
url The URL you wish to make tiny before sending.

/who

List members from your roster in various presence states. Whilst in a chat room, list room members.

Normal usage:
/who [presence]
Arguments:
presence online - Show all online contacts, includes those who are 'chat', 'away', 'xa', 'dnd'.
offline - Show all offline contacts.
chat - Show all online contacts with a presence of 'chat'.
dnd - Show all online contacts with a presence of 'dnd'.
away - Show all online contacts with a presence of 'away'.
xa - Show all online contacts with a presence of 'xa'.
dnd - Show all online contacts with a presence of 'dnd'.
available - Show all contacts with a presence of 'online', 'chat'.
unavailable - Show all contacts with a presence of 'offline', 'away', 'xa', 'dnd'.

Omitting this argument will show all contacts in your roster.
Chat room usage:
/who

List all members of the chat room.

/wins

Show a list of all currently active virtual windows in the console window, includes information about each window such as chat recipient, room, and unread messages.

Changing presence

/away

Set your status to 'away' with optional message.

Usage:
/away [message]
Arguments:
message The message you want others to see along with your presence. Omitting this argument will not set any message.

/chat

Set your status to 'chat' (availble for chat) with optional message.

Usage:
/chat [message]
Arguments:
message The message you want others to see along with your presence. Omitting this argument will not set any message.

/dnd

Set your status to 'dnd' (do not disturb) with optional message.

Usage:
/dnd [message]
Arguments:
message The message you want others to see along with your presence. Omitting this argument will not set any message.

/online

Set your status to 'online' with optional message.

Usage:
/online [message]
Arguments:
message The message you want others to see along with your presence. Omitting this argument will not set any message.

/xa

Set your status to 'xa' (extended away) with optional message.

Usage:
/xa [message]
Arguments:
message The message you want others to see along with your presence. Omitting this argument will not set any message.

Configuration

/autoping

Configure profanity to send a ping to the chat server on a regular basis. This can sometimes be required to keep a connection from closing.

Usage:
/autoping seconds
Arguments:
seconds The interval in seconds between sending pings to the server. A value of 0 will diable automatic server pings.
Default:

Set to 0 (disabled) by default.

/autoaway

Configure profanity to send either a last activity presence, or an away status change when you have been idle for a given amount of time.

If built in an X-Windows environment, with the libxss library available (this is the case with most Linux distributions), the desktop idle time will be used, otherwise the last input to the profanity window is used.

Usage:
/autoaway command value
Arguments:
command One of mode, time, message, check, see below for arguments to each.
value mode - Values are away, idle, off.
time - Time in minutes before sending presence change.
message - Optional message, or off for no message.
check - Automatically set status to online when input received, on or off.
Defaults:

mode is set to off.

time is set to 15 minutes.

message is off.

check is set to on.

Examples:

To set profanity to send your idle time (with your presence remaining online) to contacts after a period of 30 minutes, with a message of "I'm not around", and report when you are no longer idle:

/autoaway mode idle
/autoaway time 30
/autoaway message I'm not around
/autoaway check on

To set profanity to change your status to 'away' to contacts after a period of 1 hour, with no message, and not automatically update status when you are no longer idle:

/autoaway mode awat
/autoaway time 60
/autoaway message off
/autoaway check off

/beep

Set profanity to send the terminal bell signal to the terminal when a new message arrives.

On some terminals, this may result in the terminal flashing.

Usage:
/beep on|off
Default:

Set to off (disabled) by default.

/chlog

Enable chat logging. see chat logs location.

Usage:
/chlog on|off
Default:

Set to off (disabled) by default.

/flash

Set profanity to send the terminal flash signal to the terminal when a new message arrives.

On some terminals, this may result in the terminal bell sounding.

Usage:
/flash on|off
Default:

Set to off (disabled) by default.

/history

Show previous messsages with a contact when opening a new chat window.

Usage:
/history on|off
Default:

Set to off (disabled) by default.

/intype

Show when a contact is typing a message in the console, and in the profanity title bar.

Usage:
/intype on|off
Default:

Set to off (disabled) by default.

/log

Configure various system logging properties.

Usage:
/autoping setting value
Arguments:
setting Currently the only available setting is maxsize. The value must be the number of bytes the log reaches before it is rotated.
Default:

Set to 1048580 (1MB) by default.

/notify

Configure desktop notifications, profanity must be compiled with libnotify for desktop notifications to work.

Usage:
/notify type value
Arguments:
type One of message, typing, remind, see below for arguments to each.
value message - Values are on or off.
typing - Values are on or off.
remind - Seconds between showing a desktop notification reminder of unread messages.
Defaults:

message is set to off.

typing is set to off.

remind is set to 0 seconds (disabled).

Examples:

To enable desktop notifications when new messages arrive:

/notify message on

To enable desktop notifications when when a contact is typing:

/notify typing on

To get desktop notifications reminding you that you have unread messages every 30 seconds:

/notify remind 30

/outtype

Send composing chat states to contacts so they can see when you are typing. Requires /states to be on.

Usage:
/outtype on|off
Default:

Set to off (disabled) by default.

/priority

Set your presence priority associated with profanity.

Usage:
/priority value
Arguments:
value A number between -128 and 127.
Default:

Set to 0 by default.

/reconnect

Configure the interval with which profanity will try to reconnect if the server connection is lost.

Usage:
/reconnect seconds
Arguments:
seconds The interval in seconds between reconnection attempts. A value of 0 will diable automatic server reconnection attempts.
Default:

Set to 0 (disabled) by default.

/splash

Enable the ASCII logo on startup.

Usage:
/splash on|off
Default:

Set to off (disabled) by default.

/states

Enable sending of chat states to your contacts.

Chat states include notifying clients of when you are 'active', i.e. interacting with the chat window, 'inactive' when you have not interacted with the chat window after some time, and 'gone' when you have left the conversation (closed the chat window).

Usage:
/states on|off
Default:

Set to off (disabled) by default.

/theme

Choose a colour theme for profanity, see themes directory.

Usage:
/themes theme
Arguments:
theme The theme file to use. A value of default will use the default theme.
Default:

Set to default.

/titlebar

Configure what additional properties are displayed in the profanity window's title bar, and therefore visible in the desktop task bar etc.

Usage:
/titlebar property on|off
Arguments:
property The only property available is version, to show the profanity version in the title bar.
Default:

All properties are set to off.

/vercheck

Enable checking for new versions of profanity on start up, and when the /about command is called.

Usage:
/vercheck [on|off]

Calling with no argument does a check for new versions.

Default:

Set to off (disabled) by default.
